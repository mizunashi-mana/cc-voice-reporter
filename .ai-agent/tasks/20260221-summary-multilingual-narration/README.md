# サマリーの多言語対応とナレーションスタイル改善

## 目的・ゴール

Summarizer の要約内容を「Claude Code 自身が現在やっていることをストーリーを鑑みながら実況する」スタイルに改善し、多言語対応を行う。

## 背景

現在の Summarizer は「操作リストを1〜2文に要約する」という機械的なスタイル。これを「Claude Code が一人称で作業の流れを実況する」ライブコメンタリースタイルに変更することで、音声で聞いたときにより自然で状況把握しやすい案内にする。

## 実装方針

### 1. Summary の言語設定追加

- `config.ts` の `summary` セクションに `language` フィールドを追加
- `SummarizerOptions` に `language` を追加
- デフォルトは `"ja"`（後方互換性）

### 2. Summarizer のシステムプロンプト改善

**現在のプロンプト:**
- 「操作リストを日本語で1〜2文に要約してください」

**改善後のプロンプト（方向性）:**
- Claude Code が一人称で「今何をしているか」を実況するスタイル
- 操作の流れ（ストーリー）を考慮した文脈のある案内
- 言語設定に応じた出力言語の切り替え

例:
- 現在: 「ファイルを読み取り、編集しました」
- 改善後: 「設定ファイルの内容を確認して、テスト周りのコードを修正しています」

### 3. buildPrompt の改善

- 操作リストに時系列の流れがわかるようなコンテキストを付加

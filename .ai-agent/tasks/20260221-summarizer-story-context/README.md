# サマリ実況にストーリー性を持たせる

Issue: https://github.com/mizunashi-mana/cc-voice-reporter/issues/52

## 目的・ゴール

Summarizer が生成する要約に、前回の要約との繋がり（ストーリー性）を持たせる。前回からの差分や今後の流れが分かるようにすることで、音声実況として聞く際の状況把握を容易にする。

## 実装方針

1. Summarizer に前回の要約テキストを保持するフィールド (`lastSummary`) を追加
2. `callOllama` のプロンプトに前回の要約をコンテキストとして含める
3. システムプロンプトに「前回の要約からの差分を意識し、進捗の流れが分かるように要約する」旨の指示を追加
4. `buildPrompt` を拡張して前回の要約を含めるようにする

## 完了条件

- [ ] Summarizer が前回の要約テキストを保持する
- [ ] Ollama へのリクエストに前回の要約がコンテキストとして含まれる

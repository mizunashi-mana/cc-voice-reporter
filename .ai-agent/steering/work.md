# 開発ワークフロー

## 基本原則

- ユーザーとの対話的な進行: 各ステップでユーザーの確認を取る
- 早期フィードバック: 動く状態を早く作り、確認を得る
- 小さな積み重ね: 小さな単位で実装し、動作確認を繰り返す

## タスク実行フロー

### 着手前

1. タスクの README を作成（目的、方針、完了条件、作業ログ）
2. 方針をユーザーに確認

### 実装中

- ステップごとに動作確認
- lint / テストを実行して品質を維持
- 必要に応じてユーザーにフィードバックを求める

### 動作確認手順

1. `npm run build` でコンパイルエラーがないことを確認
2. `npm run lint` でリンターエラーがないことを確認
3. `npm test` でテストが通ることを確認
4. 実際に Claude Code のフックとして動作させて音声出力を確認

### 完了時

1. 完了条件を満たしているか確認
2. 作業ログを更新
3. ユーザーに完了報告
